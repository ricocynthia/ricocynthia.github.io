Problems I've solved.

```
🤝
Hawaiian Airlines Acquisition: Search API
VueNode.js BFF GoFeature FlagsAPI Design

When Alaska Airlines acquired Hawaiian Airlines, the check-in platform had to handle an entirely new class of reservations. I led the search API work that made this possible. The API needed to detect whether a confirmation code belonged to an HA reservation and redirect those guests to the HA manage site, detect migrated HA reservations (flights departing after the PSS cutoff date) and treat them as Alaska reservations, stay backwards compatible as we added last name as a required search field to support HA's API requirements, and support a new BFF v2 data gatherer for a cleaner UI response. All of it wrapped in feature flags so we could flip the switch the moment the merger was official.

👥
Group Reservation Check-In
VueTypeScript Node.js BFFGo

Group reservations (7+ passengers) had no self-service check-in path at all. I led this as my first big feature. The challenge was that groups are too large to check in all at once, so we designed an MVP where guests check in by last name. A traveler enters their confirmation code and departure airport, then provides their last name. If the reservation passes all validations, they're routed to the check-in summary page showing only passengers with that last name. Simple for the guest, complex under the hood — passenger-level authorization, validation logic, and support for both migrated and legacy reservation formats.

🌍
International Document Verification
VueNode.js GoJumiogRPC

International travelers were having to call in or go to the airport to verify their travel documents. I built and owned the self-service verification microsite end-to-end, integrating with Jumio's identity verification API, designing the backend service contracts, and making sure every edge case was handled gracefully. The result was a fully self-service experience that took real pressure off airport operations.
```
